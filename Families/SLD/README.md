# SLD Symbol Families (Single Line Diagram)

This directory holds `.rfa` Detail Item families used by `MepSymbolEngine`
on SLD drafting views. These are schematic symbols, not plan-view shapes.

**Status: empty — families generated at runtime by SymbolLibraryCreator.**

## Where families come from

Run **TAGS tab → Symbols section → Create All Symbols** in STING.
The command generates `.rfa` files from these JSON definitions:

| JSON source | Output sub-folder |
|---|---|
| `STING_SLD_SYMBOLS.json` | `_BIM_COORD/Families/Symbols/SLD/IEC` |
| `STING_SLD_SYMBOLS_IEEE.json` | `_BIM_COORD/Families/Symbols/SLD/IEEE` |
| `STING_SLD_SYMBOLS_BS.json` | `_BIM_COORD/Families/Symbols/SLD/BS` |
| `STING_SLD_SYMBOLS_NFPA.json` | `_BIM_COORD/Families/Symbols/SLD/NFPA` |
| `STING_SLD_SYMBOLS_CIBSE.json` | `_BIM_COORD/Families/Symbols/SLD/CIBSE` |

`MepSymbolEngine` searches `_BIM_COORD/Families/Symbols/` recursively so
generated families are found automatically without copying here.

Copy to this folder only if you want a static, project-independent bundle
that doesn't depend on running `Create All Symbols` first.

## Note on SLD scale

SLD drafting views have `view.Scale = 1` in Revit (not a model view).
`MepSymbolEngine` detects this and sets `Symbol Scale = 1`.
All SLD symbol geometry is authored at literal paper dimensions in the
JSON definitions (e.g. `symbolSize: 6.0` means 6 mm in model space = 6 mm plotted).
