# ── CONFIRMED PRODUCTION DEFECTS in DeliverableStateMachine ───────────────
# These tests are RIGHT and the engine is WRONG. Deliberately not fixed here:
# the remedies change how every tenant's custom workflow vocabulary is
# classified, which warrants its own PR and review rather than riding along in a
# test-health pass. Full diagnosis with file:line in docs/ROADMAP.md (DEP-14).
#
#   InferRoleByKeyword_RecognisesExtendedWorking — "BLOCKED".Contains("LOCKED")
#     is true, and the terminal scan precedes the working scan, so every
#     BLOCKED* state is misclassified as terminal.

#     already approved.

#     canonical mapping for any state named in states[]/transitions[], the
#     per-bucket merge lets the platform layer win over the project layer, and
#     an explicit roles[] block does not suppress built-in inference at runtime.
