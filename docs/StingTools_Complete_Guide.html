Introduction

What Is StingTools?

StingTools is a production-grade Revit plugin built in C# that replaces three separate pyRevit extensions (STINGDocs, STINGTags, STINGTemp) with a single compiled assembly. It delivers ISO 19650-compliant workflows for tagging, documentation, coordination, and handover across architectural, structural, MEP, and healthcare disciplines.

🏷️
ISO 19650 Tagging Engine
Automated 8-segment tag derivation across 1,106+ element categories. Real-time IUpdater auto-tagging with collision avoidance, 36 discipline containers, and TAG7 rich narrative system.
📋
BIM Management Centre
Full ISO 19650 workflow suite: BEP generation, CDE state machine, BCF 2.1 issues, document register, COBie 2.4 export (22 presets, 19 worksheets), transmittals, and 4D/5D scheduling.
📐
Drawing Template Manager
40 corporate drawing types, 11 view style packs, 199 AEC filters, scope-box binding, drift detection, and SyncStyles — all from a single JSON-driven registry with project overrides.
📄
Template Engine v1.1
MiniWord + ClosedXML pipeline for 16 embedded document templates (transmittals, TQs, RFIs, MRs, meeting minutes, handover certificates). SHA-256 tamper-evident audit chain.
🏥
Healthcare Pack
16 clinical validators, MGPS flow solver (NFPA 99), radiation shielding calculator (NCRP 147), anti-ligature audit, and 22 healthcare drawing types covering HTM/HBN/FGI standards.
⚙️
v4 MVP — Fabrication Suite

Thread-Safety Model

🔒
IExternalEventHandler pattern: All WPF button clicks route through StingCommandHandler.SetCommand(tag) → ExternalEvent.Raise() → Execute(UIApplication) on the Revit API thread. Batch operations use EscapeChecker (Win32 GetAsyncKeyState) for cancellation without blocking the API thread.

Installation & Setup

StingTools ships as a single DLL + addin manifest. A first-time project setup wizard handles shared parameter binding, template extraction, and data file seeding automatically.

Manual Installation

1
Build the assembly
Run dotnet build StingTools/StingTools.csproj -p:RevitApiPath="C:\Program Files\Autodesk\Revit 2025"
2
Copy addin manifest
Copy StingTools.addin to %APPDATA%\Autodesk\Revit\Addins\2025\ (per-user) or C:\ProgramData\Autodesk\Revit\Addins\2025\ (machine-wide)
3
Copy plugin files
Copy StingTools.dll, Newtonsoft.Json.dll, ClosedXML.dll, MiniWord.dll, Lucene.Net.dll, and the data/ folder to the assembly path.
4
Restart Revit
The STING Tools ribbon tab and dockable panel will appear. Click the STING Panel button or use View → User Interface → Dockable Panes → STING Tools.
5
Run Project Setup Wizard
Open a Revit project and click ★ Project Setup Wizard (TEMP tab). This 7-page WPF wizard handles shared parameter binding, workset creation, filters, view templates, and data file seeding.

First-Open Automatic Actions

✅
OnDocumentOpened event: StingTools automatically runs a lightweight ComplianceScan and updates the status bar with RAG tag completeness. It also calls EmbeddedTemplates.ExtractIfMissing(doc) to seed the _BIM_COORD/ folder with 16 document templates and 5 workflow JSONs — all idempotent re-opens are no-ops.

Getting Started

Quick Start — First 15 Minutes

The fastest path from a fresh install to a fully tagged, compliant BIM model.

1
Open STING Panel
Click the STING Panel button in the ribbon (or use View → User Interface → STING Tools). Dock it to the right side of Revit.
2
Run Project Setup Wizard
TEMP tab → ★ Project Setup Wizard. Complete the 7 pages (takes ~5 minutes for a typical project). This binds parameters, creates worksets and filters.
3
Auto-Populate Tokens
CREATE tab → Family-Stage Populate. This pre-fills all 7 tokens (DISC/LOC/ZONE/LVL/SYS/FUNC/PROD) from Revit spatial data without creating tags yet.
4
Tag & Combine
CREATE tab → Tag & Combine. One click: auto-detect + populate + ISO 19650 tag + write to all 36 discipline containers. Check the post-run stats dialog.
5
Validate
CREATE tab → Validate. See per-token compliance breakdown. Fix missing DISC/SYS codes and re-run. Target >80% for Green RAG status.
6
Enable Auto-Tagger (optional)
TAGS tab → Toggle Auto-Tagger. New elements placed in Revit will be auto-tagged in real time without manual intervention.

💧 Plumbing & PHE Project Workflow — Full Sequence11 steps
1 — Setup: TEMP → Master Setup. Creates MEP materials (464 types), pipe system families (DCW/DHW/HWS/SAN/RWD/GAS), and binds all PLM_* shared parameters from MR_PARAMETERS.txt.
2 — Model Fixtures: MODEL → Place Fixture. Place WCs, basins, showers, sinks, and ME equipment (calorifiers, booster sets). Each element is auto-tagged with the 9-step pipeline — DISC=P, SYS derived from the Revit pipe system connection.
3 — Route Pipework: MODEL → Create Pipe. Manually route hot and cold water, drainage, and rainwater branches. Set system type (DCW/SAN etc.) and nominal bore at placement. Or use TAGS → Auto Drop (AutoPipeDrop) for automated branch routing.
4 — Apply Templates: VIEW → Template Manager → Auto-Assign. Assigns plumb-drainage-A1-1to100 drawing type to drainage plans and mep-plan-A1-1to100 to combined services plans.
5 — Create Plumbing Filters: TEMP → Templates → Create Filters. Creates 18 plumbing ParameterFilterElement objects (DCW, DHW, SAN, RWD, GAS, medical-gas) from the AEC filter library with BS 1710 colour overrides.
6 — Tag Plumbing Elements: SELECT → Select Plumbing → Auto Tag. All fixtures and equipment receive PLM_EQP_TAG. Pipe system SYS code is inferred from GetMepSystemAwareSysCode before falling back to the category map.
7 — Sizing & Compliance: TEMP → Data Pipeline → Sizing Check. MEP Intelligence Engine runs Hardy Cross balancing and Darcy-Weisbach pressure drop. BS EN 806 / CIBSE Guide G limit checks run automatically and flag over-velocity segments.
8 — MEP Schedules: TEMP → MEP Schedules → Fixture / Equipment / System. Creates Revit schedules with connection sizes, trap types, system volumes, and all STING tag parameters for coordination review.
9 — Drawing Production: DOCS → Batch Sheets. Produces drainage plans, combined services plans, and schematic riser drawings with correct drawing type, title block, and viewport placement.
10 — Validate Slopes: TAGS → Validate Fills / Run All Validators (v4). The SlopeValidator checks every drainage branch against minimum falls (BS EN 12056-2). Failures listed by element ID and location.
11 — COBie & FM Handover: BIM → COBie Export (MEP preset). Plumbing fixtures populate the Type/Component/System/Job/Spare/Document COBie worksheets. The FM Handover Manual auto-generates the water-safety L8 section, complete with system volumes, calorifier temperatures, and deadleg schedules.

Practical walkthroughs for the most common workflows. Each guide is optimised for speed — the minimum number of steps to achieve a professional result.

Guide 1: First-Time ISO 19650 Tagging Setup

1
Project Setup Wizard
TEMP tab → ★ Project Setup Wizard. Complete all 7 pages. Estimated time: 5–10 minutes for a typical project. Focus on correct PRJ_ORG_* parameter values (project code, originator, phase).
2
Configure Tag Tokens
Open a plan view. CREATE tab → Config Editor. Set TAG_PREFIX, CATEGORY_SKIP, SEQ_SCHEME, and any CATEGORY_FORCE_SYS overrides for non-standard system types in your project.
3
Pre-Tag Audit
CREATE tab → Pre-Tag Audit. Review the predicted tag assignments for a sample view. Note any category mismatches or missing SYS codes that need CATEGORY_FORCE_SYS entries in project_config.json.
4
Family-Stage Populate
CREATE tab → Family-Stage Populate. Run across the whole project to pre-fill DISC/LOC/ZONE/LVL/SYS/FUNC/PROD. No tags are built yet — this seeds the data for step 5.
5
Tag & Combine
CREATE tab → Tag & Combine. Runs the full 9-step pipeline across the project. Watch the progress dialog for completion stats. Accept the post-run stats dialog.
6
Validate & Fix
CREATE tab → Validate. Review per-token compliance breakdown. For each red token, use the appropriate setter command (Set Discipline, Set Location, etc.) to fix outliers, then re-run Validate.
7
Enable Auto-Tagger
TAGS tab → Toggle Auto-Tagger. From now on, new elements placed in Revit are automatically tagged. No further manual runs needed for ongoing coordination.

Guide 2: Weekly ISO 19650 Data Drop

1
Run Weekly Data Drop Workflow
BIM tab → Workflow Presets → Weekly ISO 19650 Data Drop. This runs 10 steps automatically. Review any step failures in the log before proceeding.
2
Review Drawing Register
DOCS tab → Drawing Register. Verify all sheets have correct revision, suitability code, and status. Update any that were missed by the automation.
3
Create Transmittal
BIM tab → Create Transmittal (Orchestrated). Select scope: all sheets or discipline subset. The command renders B06 transmittal, starts the transmittal_default workflow, and appends to the audit log.
4
CDE Package
BIM tab → CDE Package. Creates the final package with cover sheet, transmittal, drawing register, and all PDFs. Output goes to the configured output directory.
5
Platform Upload
BIM tab → ACC Publish or Platform Sync. Uploads to Autodesk Construction Cloud / BIM 360 with ISO 19650 file naming enforced.

Guide 3: COBie Handover Preparation

1
Verify Asset Parameters
Run Completeness Dashboard (CREATE tab). Ensure all assets have PROD, SYS, FUNC, and asset-specific parameters (manufacturer, model, serial number, warranty period) populated.
2
Select COBie Preset
BIM tab → COBie Export. Select the preset matching your project type (e.g., NHS Hospital, Office, Data Centre). The preset configures all 19 COBie worksheets appropriately.

Run COBie Export

4
Generate FM Handover Manual
DOCS tab → FM Handover Manual. Generates a formatted Word document with asset register, spatial summary, system descriptions, and maintenance schedule from IoTMaintenanceCommands.
5
Issue Handover Certificate
BIM tab → Issue Deliverable → select D16 Handover Certificate template. Renders the tri-party signature block document and starts the deliverable_issue_default workflow.

Glossary

Key terms used throughout StingTools and this guide.

TermDefinition
ASS_TAG_1The primary 8-segment ISO 19650 tag parameter. The universal container written by all tagging commands.
BCF 2.1BIM Collaboration Format — an open standard for exchanging model-based issues between BIM tools.
BEPBIM Execution Plan — the project document defining how BIM will be used per ISO 19650-2.
CDECommon Data Environment — the shared repository where project information is managed per ISO 19650.
COBie 2.4Construction Operations Building Information Exchange — the handover data standard for FM teams.
DISCDiscipline code — M (Mechanical), E (Electrical), P (Plumbing/Public Health), A (Architectural), S (Structural), FP (Fire Protection), LV (Low Voltage/Comms), G (General).
DrawingTypeA JSON bundle defining all presentation parameters for one drawing type — scale, title block, view template, crop strategy, slots, annotation rules.
IUpdaterA Revit API interface that triggers a callback when elements change. StingAutoTagger uses it for real-time auto-tagging.
ISO 19650International standard for managing information over the whole life cycle of a built asset using BIM.
LODLevel of Development — a measure of the completeness and reliability of BIM information (LOD 200–500).
MaxRectsA bin packing algorithm (Best Short Side Fit heuristic) used by the Sheet Manager for optimal viewport placement.
TermDefinition
MGPSMedical Gas Pipeline System — the piped gas supply in healthcare buildings (O₂, N₂O, Medical Air, Vacuum, CO₂).
MIDPMaster Information Delivery Plan — the project-level schedule of BIM deliverables per ISO 19650-2.
MIMModel Information Management — the Planscape add-on for FM digital twin and asset lifecycle tracking.
NLPNatural Language Processing — the TAGS Studio text-input engine that maps plain English to STING commands.
RAGRed/Amber/Green status indicator — Red <50%, Amber 50–80%, Green >80% compliance.
RDSRoom Data Sheet — a document defining all technical requirements for a specific clinical room.
SEQ4-digit zero-padded sequence number (0001–9999) — the unique identifier within a (DISC, SYS, LVL) group.
Suitability CodeISO 19650 code indicating the document's purpose — S0 (WIP), S2 (Shared), S3 (Review), S4 (Published).
TAG7The rich descriptive narrative parameter system — 6 sub-sections (A–F) covering identity, system, spatial, lifecycle, technical, and classification data.
TransactionGroupA Revit API mechanism grouping multiple Transactions — used by workflow presets for atomic rollback on failure.
ViewStylePackA named set of graphic overrides, filters, and style references shared across multiple DrawingTypes to ensure graphic consistency.
WorksharingRevit's multi-user collaboration model using worksets and a central file — StingAutoTagger respects workset ownership before auto-tagging.
