How the Auto-Tagger Works

The StingAutoTagger is a background Revit IUpdater — a type of event listener built into the Revit API. Here is the exact sequence when you place an element:

1
Element Placed
You place a mechanical fan coil unit in Revit. The IUpdater fires on GetChangeTypeElementAddition(). This happens in the same transaction as the placement — no delay.
2
Token Auto-Detection (5 Layers)
The TokenAutoPopulator runs through 5 detection layers for each token:
  • DISC: Revit category → discipline lookup table → M (Mechanical)
  • LOC: Spatial containment → linked model scope → BLD1
  • ZONE: Room boundary test → zone parameter → grid region → Z01
  • LVL: Host level → elevation mapping → L02
  • SYS: MEP system name → family parameter → HVAC
  • FUNC: Family type parameter → system function → SUP
  • PROD: Family name pattern matching against 1,200-entry lookup → FCU
3
Tag Assembly & Sequence
Tokens are combined into the ISO format: M-BLD1-Z01-L02-HVAC-SUP-FCU-0001. The SEQ counter reads from the .sting_seq.json sidecar file and auto-increments. Collision detection checks all 10,000 existing tags in O(1) time using a HashSet index.
4
Write to 36 Containers
The completed tag is written to ASS_TAG_1_TXT (the primary ISO tag) plus the relevant discipline-specific containers (e.g. M_TAG_TXT for Mechanical). Only the 6–10 containers applicable to the element's category are written — not all 36 — for performance.
5
TAG7 Narrative & Visual Tag (Optional)
If TAG7 is enabled, a human-readable narrative is assembled: "Mechanical HVAC Fan Coil Unit on Level 2, Zone 1, Building 1". If visual tagging is on, a Revit IndependentTag annotation is placed at the optimal collision-free position around the element.

Data Storage — The _bim_manager Folder

All BIM management data is stored locally in a _bim_manager/ folder placed automatically alongside your .rvt file. This makes data portable — the folder travels with the model.

ℹ
Key files in _bim_manager/: issues.json (issue register), documents.json (document register), transmittals.json, snapshots/ (revision tag snapshots), .sting_seq.json (SEQ counters), .sting_warnings_baseline.json (warnings trend), .sting_compliance_trend.json (compliance history).

Thread Safety & Stability

All commands — whether clicked in the dock panel or triggered by a workflow preset — are dispatched through a Revit IExternalEventHandler. This guarantees that all Revit API calls execute on the main Revit thread, preventing crashes and ensuring worksharing compatibility. StingTools is a compiled C# plugin, not a script — it is stable, versioned, and does not require Python or pyRevit.

Tagging Walkthrough

A complete walkthrough of the tagging system — from first-time batch tagging to incremental updates, compliance validation, and visual annotation placement.

First-Time Tagging: Full Model

1
Run a Pre-Tag Audit (Recommended)
A dry run that predicts tag assignments without writing anything. The result panel shows: elements to be tagged per discipline, which tokens will fall back to placeholders, predicted collision count, and spatial detection coverage.CREATE → Pre-Tag Audit
2
Run Tag & Combine
The flagship one-click command: populate all tokens → assemble ISO tag → write all discipline containers → resolve SEQ collisions → report. A progress dialog shows ETA. Press Escape to cancel safely at any point.CREATE → Tag & Combine
3
Validate Tags — Check Compliance
Run Validate Tags for a 4-bucket compliance report: RESOLVED (green, all 8 segments valid) / COMPLETE_PLACEHOLDERS (amber) / INCOMPLETE (orange) / UNTAGGED (red). Results broken down by discipline, level, and category.CREATE → Validate Tags
4
Resolve All Issues — One Click
Fixes all INCOMPLETE and most PLACEHOLDER tags automatically. Uses spatial re-detection with looser boundaries, type-level token inheritance, nearest-neighbour token copy, and generic code fallback where specific codes can't be determined.CREATE → Resolve All Issues
5
Review the Tag Completeness Dashboard
Visual breakdown of tag coverage by discipline, level, category, and zone. Export as HTML or CSV for client reporting or BEP milestone records.CREATE → Dashboard

Ongoing: Daily Incremental Tagging

1
Tag New Only
Skips already-tagged elements and processes only new additions. 10–100× faster than a full batch tag. Safe to run at any time without overwriting existing tags.CREATE → Tag New Only
2
Retag Stale Elements
Elements whose geometry changes are automatically flagged as stale (STING_STALE_BOOL = 1) by the background StingStaleMarker. Run Retag Stale to refresh only those elements.CREATE → Retag Stale

Handover & COBie Export

Producing a complete FM-ready handover package — COBie V2.4, O&M manuals, asset registers, and maintenance schedules — directly from your tagged Revit model in under 30 minutes.

ℹ
COBie (Construction Operations Building information exchange) is the standard format for transferring built asset data to facilities management. StingTools produces fully validated COBie V2.4 spreadsheets from your tagged model. What used to take 3 days now takes under 15 minutes.

Pre-Handover Validation

1
Run the Handover Readiness Preset
From BIM tab Quick Workflows, click Handover. This 9-step preset validates tag completeness, checks all required COBie fields are populated, resolves outstanding issues, and generates the handover compliance report.BIM → Quick Workflows → Handover

Review COBie Readiness Report
The report shows which of the 19 COBie worksheets have complete data and which have gaps. Common gaps: missing Warranty data, incomplete Space names, and unassigned Type classifications. Fix gaps before full export.

Reference

Glossary & Key Terms

Definitions of key terms, abbreviations, and concepts used throughout StingTools and ISO 19650 BIM workflows.

TermDefinition
ASS_TAG_1_TXTThe primary ISO 19650 asset tag parameter stored on each Revit element. Contains the full 8-segment tag string. The definitive identifier for any asset in the model.
Auto-TaggerThe background Revit IUpdater that detects and assigns ISO tags to newly placed elements in real time. Toggle via CREATE tab. Processes 22 element categories with discipline filter and worksharing safety checks.
BEPBIM Execution Plan. A formal document defining how BIM will be implemented on a specific project. Managed via the 7-page BEP Wizard in the BIM tab.
BCF 2.1BIM Collaboration Format. Open standard for exchanging issues and clashes between BIM platforms (Revit, Navisworks, Solibri, BIMcollab).
CDECommon Data Environment. The ISO 19650 information lifecycle with four states: Work In Progress → Shared → Published → Archived. Managed via Document Management in BIM tab.
COBieConstruction Operations Building information exchange. V2.4 is the current standard for transferring asset data to FM teams at handover. 19-worksheet spreadsheet format.
DISCDiscipline token. First segment of the ISO tag. Key codes: M (Mechanical), E (Electrical), P (Plumbing), A (Architectural), S (Structural), FP (Fire Protection), LV (Low Voltage / Security), G (General).
ISO 19650International standard for managing information over the whole life cycle of a built asset using BIM. StingTools implements this standard natively.
Stale FlagWhen an element's geometry changes after tagging, STING_STALE_BOOL = 1 is set automatically by the background StingStaleMarker. Cleared by running Retag Stale.
Tag ContainerA discipline-specific shared parameter holding a formatted version of the ISO tag for that discipline's FM or scheduling system. StingTools manages 36 containers across 22 disciplines in one write operation.
TAG7The rich narrative parameter. Contains a human-readable description of the asset assembled from all token values: e.g. "Mechanical HVAC Air Handling Unit on Level 2, Zone 1, Building 1 — Ref: M-BLD1-Z01-L02-HVAC-SUP-AHU-0003".
TokenOne of the 8 building blocks of an ISO 19650 tag: DISC, LOC, ZONE, LVL, SYS, FUNC, PROD, SEQ. All 8 are auto-detected by StingTools using a 5-layer spatial intelligence algorithm.
StingMIMThe optional Maintenance Information Management module. Adds FM operations capabilities to the StingBIM platform: asset registry, digital twin sync, IoT sensor integration, maintenance scheduling, and O&M manual generation.
Workflow PresetA JSON-defined sequence of StingTools commands that execute in order with optional conditional logic. Built-in presets cover all major coordination cycles. Custom presets can be created without coding.
_bim_manager/The data folder created by StingTools alongside the .rvt file. Contains all BIM management data: issues.json, documents.json, transmittals.json, snapshots/, .sting_seq.json, compliance trend data, and warning baselines.
